# Веб-версия проекта (миграция бота → сайт)

Причина: сервер в РФ режет доступ к Telegram (см. дневник). Сайт убирает зависимость от
Telegram — файлы грузятся в браузере напрямую на сервер, вся ИИ-логика переиспользуется.

Бот заморожен тегом `bot-v1.0` и веткой `bot-legacy` — код цел, разворачивается в любой момент.

## Стек
- **Backend**: FastAPI (Python), async SQLAlchemy (та же БД/модели).
- **Frontend**: React + Vite + TypeScript (SPA).
- **Переиспользуем как есть**: `bot/services/*` (расшифровка, роли, анализ блокнота, отчёты,
  ИПР, библиотеки, чтение заполненного блокнота, чанкование), `bot/models/*`, `bot/resources/*`.
- **Заменяем**: aiogram-хендлеры, `media_jobs`, `telegram_files` → REST + загрузка файлов +
  фоновая обработка.

## Структура
```
web/
  backend/
    app.py            # FastAPI-приложение
    deps.py           # зависимости (DB-сессия, auth)
    schemas.py        # pydantic-схемы запросов/ответов
    jobs.py           # фоновая обработка (расшифровка/анализ) — по образцу media_jobs
    routers/
      assessments.py  # центры / участники / упражнения (CRUD)
      files.py        # загрузка инструкций / аудио / заполненного блокнота
      reports.py      # запуск обработки, статус, скачивание отчёта/ИПР/блокнота
  frontend/           # React + Vite + TS (SPA)
```

## Фазы
1. **Backend-каркас** (текущая): FastAPI + БД + CRUD центров/участников/упражнений. ← начато
2. **Загрузка файлов**: инструкции (pdf/docx), аудио, заполненный блокнот — на диск + записи в БД.
3. **Обработка**: фоновые задачи (аудио → расшифровка → роли → анализ → блокнот) и путь
   «загружен заполненный блокнот». Статус через опрос.
4. **Отчёт/ИПР**: генерация и скачивание (переиспуем `reports`, `report_pptx`, `development_advice`).
5. **Frontend SPA**: дашборд → шаги упражнения → страница результата.
6. **Auth**: логин (пока простой пароль, как `ADMIN_BOT_PASSWORD`).
7. **Деплой**: uvicorn за nginx, домен + HTTPS.

## Заметки
- `bot.config` теперь не требует `BOT_TOKEN`/Yandex (дефолты `""`), чтобы веб импортировал сервисы.
- Расшифровка идёт через AI Tunnel (канал к нему с сервера жив — Telegram отдельно).
- БД та же (SQLite/Alembic); модели переиспользуются, миграции продолжают действовать.
