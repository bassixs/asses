# Как пользоваться ботом

Это короткая инструкция для HR/ассессора без технических деталей.

## Что делает бот

Бот помогает обработать упражнение ассессмент-центра:

```text
аудио -> расшифровка -> роли -> оценка индикаторов -> заполненный Excel -> отчет -> ИПР
```

Бот различает две роли:

- `Участник` - оцениваемый человек, чьи компетенции проверяются;
- `Ведущий` - все остальные: ассессор, наблюдатель, ролевой сотрудник, клиент, интервьюер.

Важно: в ролевой встрече ролевой сотрудник может много говорить, но он не оцениваемый участник.

## Быстрый тест одной записи

1. Отправьте боту аудио.
2. Выберите движок расшифровки:
   - `AI Tunnel Whisper` - основной тестовый вариант;
   - `Yandex` - резервный вариант;
   - `NeuroAPI Whisper` - тестовый вариант, может быть нестабилен.
3. Дождитесь сообщения:

```text
Задача #N: расшифровано через ...
ID записи: X
```

4. Нажмите кнопку `Скачать расшифровку`, чтобы проверить текст и роли.
5. Нажмите `Оценить компетенции` или отправьте:

```text
/assess X
```

Оценка запускается в фоне. Бот пришлет результат отдельным сообщением.

## Полный сценарий ассессмент-центра

### 1. Создать ассессмент-центр

```text
/create_center Тест Беседа с сотрудником
```

Бот вернет ID центра, например `#1`.

### 2. Добавить оцениваемого участника

```text
/add_participant 1 Маргарита
```

Где `1` - ID центра.

Бот вернет ID участника, например `#1`.

### 3. Добавить упражнение

```text
/add_exercise 1 Беседа с сотрудником
```

Где `1` - ID участника.

Бот вернет ID упражнения, например `#1`.

### 4. Отправить аудио упражнения

Отправьте аудиофайл в Telegram.

Бот попросит выбрать STT-движок. Для текущих тестов обычно выбираем:

```text
AI Tunnel Whisper
```

После завершения бот вернет ID записи, например:

```text
ID записи: 2
```

### 5. Привязать запись к упражнению

```text
/attach_record 1 2
```

Где:

- `1` - ID упражнения;
- `2` - ID записи.

После привязки бот автоматически переуточнит роли с учетом имени оцениваемого участника.

### 6. Проверить расшифровку

Нажмите кнопку:

```text
Скачать расшифровку
```

Или выполните:

```text
/rebuild_transcript 2
```

`/rebuild_transcript` пересобирает роли от сырой расшифровки, если запись уже привязана к упражнению.

### 7. Отправить Excel-блокнот наблюдателя

Отправьте `.xlsx` файл блокнота.

Бот вернет ID блокнота, например:

```text
ID блокнота: 3
```

### 8. Привязать блокнот к упражнению

```text
/attach_notebook 1 3
```

Где:

- `1` - ID упражнения;
- `3` - ID блокнота.

### 9. Обработать упражнение

Вариант через workflow:

```text
/process_exercise 1
```

Где `1` - ID упражнения.

Бот заполнит Excel-блокнот и пришлет готовый файл.

Альтернативный вариант без workflow:

```text
/fill_notebook 2 3
```

Где:

- `2` - ID записи;
- `3` - ID блокнота.

`/fill_notebook` запускается в фоне и пришлет готовый Excel отдельным сообщением.

### 10. Сформировать отчет участника

Когда все упражнения участника обработаны:

```text
/generate_report 1
```

Где `1` - ID участника.

Бот пришлет `.docx` отчет.

### 11. Сформировать ИПР

Сначала должен быть сформирован отчет.

Потом:

```text
/generate_ipr 1
```

Бот пришлет `.docx` индивидуальный план развития.

## Полезные команды

Показать центры:

```text
/centers
```

Показать участников центра:

```text
/participants 1
```

Показать упражнения участника:

```text
/exercises 1
```

Показать критерии:

```text
/criteria
```

Показать последние записи:

```text
/my_records
```

Показать оценки:

```text
/my_assessments
```

Пересобрать расшифровку:

```text
/rebuild_transcript 2
```

## Админка

Открыть:

```text
/admin
```

Пароль:

```text
1172
```

В админке можно посмотреть статистику и удалить записи, блокноты, оценки или все данные.

Важно: админка пока простая. В продакшене нужно добавить список разрешенных администраторов и более строгую защиту.

## Что проверять в тестах

После расшифровки:

- текст должен быть полным, без краткого summary;
- роли должны быть только `Ведущий` и `Участник`;
- `Участник` должен быть именно оцениваемый человек;
- ролевой сотрудник должен попадать в `Ведущий`;
- если роли ошиблись, используйте `/attach_record`, затем `/rebuild_transcript`.

После заполнения блокнота:

- каждый `+` должен иметь цитату;
- `-` не должен иметь выдуманную цитату;
- `НЗ` должен ставиться только когда индикатор объективно не мог наблюдаться;
- уровень компетенции должен считаться без учета `НЗ`.

## Текущие ограничения

- Для AI Tunnel/NeuroAPI сейчас есть сжатие аудио под лимит, но нет полноценной нарезки длинных файлов на части.
- `/process_exercise` пока выполняется прямо в handler, поэтому может отвечать дольше, чем `/fill_notebook`.
- DOCX отчет и ИПР уже создаются, но еще не идеально повторяют финальные шаблоны заказчика.
- PDF-export пока не реализован.
- Админка пока защищена только паролем.
