# Статус проекта и дорожная карта

Единый «блокнот» по состоянию проекта: что сделано, что живёт на сервере, что предстоит.
Обновлено: 2026-07-05.

---

## 1. Короткая картина

Проект — обработка упражнений ассессмент-центра: аудио → расшифровка → роли → оценка
поведенческих индикаторов → заполнение блокнота наблюдателя → отчёт участника + ИПР.

**Сейчас идёт пивот: Telegram-бот → веб-сайт.** Причина — сервер (Москва) не может достучаться
до серверов Telegram (провайдер режет доступ к Telegram; общий интернет работает). Из-за этого у
бота ломались все операции с файлами. Сайт убирает Telegram полностью: файлы грузятся в браузере.

- **Бот** собран, работает как продукт, **заморожен** (тег `bot-v1.0`) — можно вернуть в любой момент.
- **Сайт** — бэкенд и фронтенд готовы, **развёрнут и живой** на сервере (пока временно).

---

## 2. Почему пивот (корень проблемы)

Диагностировано по логам контейнера `telegram-bot-api`:
- `getUpdates` (сообщения) кое-как работает через одно долгоживущее соединение;
- `getFile`/скачивание файлов — виснет: соединения к дата-центрам Telegram (в т.ч. файловому DC)
  либо не устанавливаются, либо крайне медленно;
- прямая проверка: **все IP Telegram недоступны** с сервера (таймаут 45с), а обычный интернет
  (8.8.8.8, github, AI Tunnel) — мгновенно.

Вывод: **провайдер блокирует/душит Telegram** (типично для РФ, усилилось в начале июля). В коде это
не чинится. Отключение IPv6 и рестарты не помогли (заблокирован и IPv4). Правильные пути: прокси/VPN
к Telegram, перенос сервера — **или уход от Telegram = сайт** (выбрано).

## 3. Что заморожено — бот `bot-v1.0`

Полнофункциональный бот (ветка `bot-legacy`, тег `bot-v1.0`). Умеет:
- гайд-флоу кнопками: центр → участник (приватный код) → упражнение → инструкции → аудио/блокнот;
- STT через AI Tunnel Whisper (+ чанкование по размеру и длительности);
- разметка ролей (ведущий/участник) с учётом инструкций упражнения;
- анализ блокнота, статусы +/−/НЗ, уровни по формуле ТЗ, таймкоды цитат, проверка цитат;
- **4 формата упражнений** проверены (Беседа, Планирование, Саморазвитие, Аналитический кейс);
- **групповое упражнение** + загрузка уже заполненного блокнота;
- отчёт DOCX/PPTX + ИПР, библиотека компетенций/упражнений, сводный отчёт по нескольким упражнениям;
- устойчивость к подвисаниям (ретраи скачивания/отправки, мягкие уведомления).

Восстановить бота: `git checkout bot-v1.0` → развернуть как раньше (нужен рабочий канал к Telegram).

## 4. Сайт — что готово

Ветка **`web`**. Переиспользует всё ИИ-ядро бота без изменений: `bot/services/*`, `bot/models/*`,
`bot/resources/*`. Заменён только слой доставки (Telegram → REST + загрузка файлов).

### Бэкенд (FastAPI) — `web/backend/`
- `app.py` — приложение, HTTP Basic-авторизация, раздача собранного SPA.
- `deps.py` — сессия БД, sentinel-владелец для веб-строк.
- `schemas.py` — pydantic-схемы.
- `processing.py` — конвейер (расшифровка→роли→анализ→блокнот, чтение заполненного, сборка
  отчёта+LLM-персонализация, рендер DOCX/PPTX/ИПР).
- `jobs.py` — фоновая обработка аудио + статус.
- `routers/` — `assessments` (CRUD), `files` (загрузки), `reports` (статус, отчёт, ИПР, скачивания).
- **17+ эндпоинтов**, документация — `/docs`.

### Фронтенд (React + Vite + TypeScript) — `web/frontend/`
- SPA: дашборд → центр → участник → упражнение; страница результата; скачивания.
- Шаг упражнения повторяет гайд-флоу (инструкции, выбор способа, загрузки, опрос статуса).

### Проверено на сервере (смоук-тест, реальные данные)
- CRUD, загрузка **заполненного блокнота** (уровни читаются из правильных ячеек — подтверждено),
  агрегация, **вызовы LLM к AI Tunnel**, генерация **DOCX-отчёта (48 КБ)** и **ИПР (41 КБ)** — всё 200.
- Фронтенд собирается чисто (`tsc && vite build`), развёрнут, отвечает снаружи.

- **БД:** `/opt/asses/data/app.db` (общая), миграции применены. Загрузки — `/opt/asses/data/uploads`,
  результаты — `/opt/asses/data/reports`.
- **Доступ к сайту:** http://85.239.35.73:8000 — Basic-auth, логин любой, пароль = `ADMIN_BOT_PASSWORD` (`1172`).
- **Firewall:** ufw активен, порт `8000/tcp` открыт.
- **Прочее:** IPv6 отключён (`/etc/sysctl.d/99-disable-ipv6.conf`); cron рестарта `telegram-bot-api`
  каждые 6ч (наследие бота, для сайта не нужен).

### Перезапустить сайт вручную
```bash
cd /opt/asses-web && git fetch -q origin web && git reset --hard origin/web
# если менялся фронтенд: cd web/frontend && npm install && npm run build
set -a; . /opt/asses/.env; set +a
export DATABASE_URL="sqlite+aiosqlite:////opt/asses/data/app.db"
export DOWNLOAD_DIR="/opt/asses/data/uploads"
pkill -f "uvicorn web.backend.app"; sleep 1
cd /opt/asses-web
nohup /opt/asses/.venv/bin/uvicorn web.backend.app:app --host 0.0.0.0 --port 8000 >/var/log/asses-web.log 2>&1 &
```

- [ ] Выделенная БД/каталоги для сайта (или осознанно оставить общие с ботом).

### Функционал / доводка

- [ ] Авторизация: сейчас HTTP Basic (один пароль). При желании — страница логина/пользователи/роли.
- [ ] UI-полировка: индикаторы загрузки, обработка ошибок, список/удаление, множественные инструкции,
      выбор формата отчёта на странице, мобильная вёрстка.
- [ ] Прогресс обработки аудио сейчас в памяти (один процесс) — при нескольких воркерах/рестарте
      теряется; при масштабировании вынести в БД/очередь.

### Решения
- [ ] Судьба бота: оставить замороженным / удалить сервис / вернуть, если появится канал к Telegram.
- [ ] Влить `web` в `main`, когда сайт станет основным продуктом.

## 8. Ключевые заметки

- Вся ИИ-логика (STT, роли, анализ, отчёты, ИПР, библиотеки) — **общая**, живёт в `bot/services`
  и переиспользуется и ботом, и сайтом. Меняем логику один раз — работает в обоих.
- `bot/config.py`: бот-специфичные креды (`BOT_TOKEN`, Yandex) сделаны необязательными (дефолт `""`),
  чтобы сайт импортировал сервисы без них.
- Расшифровка и LLM идут через **AI Tunnel** — канал к нему с сервера рабочий (в отличие от Telegram).
- Приватность: участник хранится по коду/номеру, без ФИО.
